The Solus — one internal platform that pulls scattered export operations into a single place.

When I arrived at HQ there was no standard playbook for export operations — knowledge lived in people's heads, forecasts were checked by hand, and product setup was slow and error-prone. I designed and built four tools to fix that, then unified them under one roof so ~200 staff could reach everything from a single entry point.

Every piece was built within a locked-down corporate environment — no external automation tools, no outside cloud — using only approved software. Tap any tool below to open a working demo with mock data.

Operations knowledge base

A searchable internal manual covering eight task categories, each mapped to a clear responsibility matrix so every team knows who owns what. It auto-updates from the source, replacing tribal knowledge with a single source of truth — and cutting the repeat questions that used to fill inboxes.

One Globe — 3D trade-flow

An interactive globe that maps the food division's import and export flows by product group and volume. A spreadsheet of country-pair tonnage is hard to feel; rendered as arcs on a rotating globe, the concentration of trade — and the gaps — become obvious in seconds. Built for leadership reviews where a chart needs to land instantly.

Forecast accuracy engine

An Excel-based toolkit that screens ~8,000 demand-plan records each month, auto-flagging any figure that deviates 30%+ from its 3-month rolling average or prior-year baseline. Flagged items trigger an email to the owner for review. What used to be slow manual eyeballing is now an exception-only workflow — attention goes where it's actually needed.

Excel · Power Query Anomaly detection Auto-alerts
04Open demo →

Material registration & code mapping

Registering a new export material used to mean two manual jobs: keying data into the MDG web form, then mapping domestic codes to export codes inside SAP — together about 40 minutes per item and a steady source of typos. I chained both with Power Automate Desktop driving the MDG form straight from an Excel source, and a SAP GUI script handling the code lookup and mapping. One run now does in about five minutes what used to take forty across two systems.
